# Network Operations Reporting & Analytics — Snowflake Demo App

A multi-persona React + Express demo app for telecom network operations teams, showcasing Snowflake's AI and analytics capabilities across four roles: Network Engineer, Performance Analyst, Network Manager, and Executive.

Built on Snowflake's [certified telecom NOC solution guide](https://www.snowflake.com/en/developers/guides/telecom-network-ops-intelligence/).

## Quick Start

### 1. Load the Data

Run the setup script in a Snowflake SQL Worksheet. It creates the `NETWORK_OPERATIONS` database and automatically loads all data from GitHub:

```sql
-- In Snowsight: Projects > Workspaces > New SQL File
-- Paste and run scripts/setup.sql
```

This creates:
- `NETWORK_OPERATIONS` database with schemas: `RAN_4G`, `RAN_5G`, `CORE_4G`, `CORE_5G`, `TRANSPORT`, `ANALYTICS`, `STAGING`, `AI`
- Dimension tables: `DIM_CELL_SITE` (450 rows), `DIM_NETWORK_ELEMENT` (480 rows)
- Fact tables: `FACT_RAN_PERFORMANCE` (604,800 rows), `FACT_CORE_PERFORMANCE` (13,440 rows)
- Synthetic ops tables: `ALARMS`, `INCIDENTS`, `SLA_TARGETS`, `TEAMS`, `ENGINEERS`
- AI audit tables: `AI.TRIAGE_RUNS`, `AI.ASSIST_OUTPUTS`

### 2. Create the Semantic View & Agent

```sql
-- Run scripts/semantic-view.sql
-- Run scripts/agent.sql
```

### 3. Run Locally

```bash
cd app
npm install
cp .env.example .env   # fill in your Snowflake connection details
npm run dev
```

App runs at `http://localhost:3000`.

### 4. Deploy to SPCS

```bash
# Build and push Docker image
docker build -t network-ops-app ./app
docker tag network-ops-app <your-snowflake-registry>/network-ops-app:latest
docker push <your-snowflake-registry>/network-ops-app:latest

# Deploy service (run in Snowflake)
-- scripts/deploy-spcs.sql
```

## Data Model

Data originates from Snowflake's certified guide ([sfguide-telecom-network-operations-analytics](https://github.com/Snowflake-Labs/sfguide-telecom-network-operations-analytics)) — 450 cell sites across Portugal (Lisboa, Porto, Braga, Coimbra, Faro), September 2025 time window, 15-minute measurement intervals.

Additional synthetic tables (ALARMS, INCIDENTS, SLA_TARGETS) are generated by `scripts/synthetic-data.sql`.
